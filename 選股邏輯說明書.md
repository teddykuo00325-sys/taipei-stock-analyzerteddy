## 🧮 二、主評分系統（11 維度 / -100 ~ +100）

每檔股票會被 11 個維度評分，加總後得到主分數。**朱家泓派預設權重**：

| # | 維度 | 加分情境 | 減分情境 | 權重 |
|---|------|---------|---------|------|
| 1 | 📐 **均線排列** | 多頭排列 +30 / 偏多 +15 | 偏空 -15 / 空頭 -30 | ±30 |
| 2 | 🌊 **波浪位置** | 第 3 波上漲 +15 | 第 5 波末端 -5 / 第 3 波下跌 -15 | ±15 |
| 3 | 📏 **葛蘭碧八法** | #1 突破 +12 / #2 回測 +8 | #5 跌破 -12 / #6 反彈賣 -8 | ±15 |
| 4 | 🏦 **三大法人** | 連續買超大量 +10 | 連續賣超大量 -10 | ±10 |
| 5 | 💰 **融資融券**（5 維度子系統） | 資減價漲、券資比高 | 資增價跌、融資使用率過高 | ±15 |
| 6 | 🔬 **計量物理** | Hurst > 0.55（趨勢）+5 | Hurst < 0.45（隨機）-5 | ±10 |
| 7 | 🌀 **黃金切割** | 接近 38.2/61.8% 支撐 +8 | 接近 61.8% 壓力 -8 | ±10 |
| 8 | 📅 **週線方向** | 週線多頭 +8 | 週線空頭 -8 | ±8 |
| 9 | 🚨 **進出場訊號** | 朱式大量突破 +12 / 帶量突破 +8 | 跌破訊號同等扣 | ±15 |
| 10 | 🕯️ **K 線型態** | 紅 K 吞噬 / 三紅兵 +6 each | 黑 K 吞噬 / 黃昏星 -6 | ±12 |
| 11 | 📐 **型態學** | W 底 / 頭肩底 +12 | M 頭 / 頭肩頂 -12 | ±12 |

加總後 clip 到 ±100。

### 分數對應評估

```
分數 ≥ +60   → 多方   → 強力買進
分數 +30~+60 → 偏多   → 買進
分數 -30~+30 → 中立   → 觀望
分數 -60~-30 → 偏空   → 賣出
分數 ≤ -60   → 空方   → 強力賣出
```

## 🥇 三、Tiebreaker 同分排序（v2，regime-aware）

當多檔股票主分數相同（譬如 5+ 檔都 100 分），需要二級排序挑「短線 3 天勝率高」的標的。

### 7 個正向維度

| # | 維度 | 觸發條件 |
|---|------|---------|
| F | **甜蜜起漲** ★ | 昨日漲幅 +1~3% |
| D | **不過熱** ★ | RSI < 70 + 乖離(收/MA10) < +5% |
| B | **法人連續買超** | 三大法人 ≥ 5000 張 |
| A | 爆量突破 | 量 ≥ 1.5×VMA20 + 突破 10 日新高 |
| G | 軋空潛力 | 券資比 > 15% |
| C | 動能加速 | 5MA/10MA ≥ 1.02 + 10MA/20MA ≥ 1.01 |
| E | KD/MACD 鮮度 | KD 金叉 ≤ 3 日 + Hist 連續放大 |

★ = 經過 30 天 27,741 觸發點實證，勝率 54%+ 最強的兩個維度

### 5 個殺手扣分

| 條件 | 扣分 |
|------|------|
| 乖離 > +8% | -15 |
| 昨日漲幅 > +7%（追高） | -10 |
| 連續 5+ 日紅 K | -10 |
| 出現吊人/流星/長黑/黃昏星 | -10 |
| 法人轉賣 > 500 張 | -8 |

### Regime-aware 權重切換（自動）

系統根據 TWII MA20 vs MA60 差距自動偵測 regime，並動態調整 7 維度權重：

| Regime | 條件 | 重視 | 邏輯 |
|--------|------|------|------|
| 🔴 強多頭 | MA20-MA60 ≥ +3% | F 甜蜜+25、D 不過熱+20 | 不追熱比追熱贏 |
| 🟢 強空頭 | MA20-MA60 ≤ -3% | A 爆量+20、C 動能+18 | 反彈時動能=真實力 |
| ⚪ 整理 | ±3% 內 | 折衷權重 | 多空中性 |

## 🚦 四、5 層智慧過濾（提升勝率）

### Lv1 大盤 Regime 過濾（硬性）

| 大盤狀態 | 多單 | 空單 | 資金規模 |
|---------|------|------|---------|
| 🔴 強多頭 | ✅ 允許 | ❌ **禁開** | 100% |
| 🟢 強空頭 | ❌ **禁開** | ✅ 允許 | 100% |
| ⚪ 整理 | ✅ 允許 | ✅ 允許 | ×50% |

### Lv2 絕對分數門檻

- 做多：**必須 ≥ +70**（朱家泓「強力買進」）
- 做空：**必須 ≤ -70**（朱家泓「強力賣出」）
- 不達標一律剔除

### Lv3 產業分散

- 同產業最多 **2 檔**
- 防止「全押航運」這種集中風險

### Lv4 動態持有期 + 技術停損

| Regime gap | 建議持有 |
|-----------|---------|
| ≥ 5%（強趨勢） | **10 天** |
| 3-5%（中度） | **7 天** |
| < 3%（整理） | **3 天** |

**MA10 強制停損**：
- 多單跌破 MA10 → 立即結算
- 空單突破 MA10 → 立即回補

### Lv5 訊號構成過濾

**做多剔除**：
- 波浪在「上升 5 波末端」（追高風險）
- 葛蘭碧顯示賣出訊號（#5~#8）

**做空剔除**：
- 波浪在「下跌 5 波末端」（殺低反彈風險）
- 葛蘭碧顯示買進訊號（#1~#4）

## 🔑 八、設計哲學

1. **多訊號融合，不靠單一指標**：11 個維度合成主分數，降低單一假訊號的影響。
2. **Regime-aware**：同一個技術指標在多頭/空頭/整理應該有不同權重 — 不是「永遠重視某指標」。
3. **可解釋性**：每個維度都能告訴你「為什麼這檔被選」，不是黑盒模型。
4. **保守過濾**：寧可錯過、不要錯買 — 5 層過濾優先擋掉爛交易，再挑好交易。
5. **歷史實證調校**：tiebreaker 權重經過 30 天回測，不是憑感覺設的。
6. **fail-soft**：任何資料源失敗時都有 fallback，整體系統不會因單一 API 掛掉而完全失效。
